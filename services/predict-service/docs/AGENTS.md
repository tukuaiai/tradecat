# docs/

本目录存放项目文档与调研材料；目标是让新读者在 5 分钟内看清：系统是什么、为什么这么设计、下一步改哪里。

```
docs/
├── AGENTS.md
├── architecture.md                    # 项目总体架构概览（高层）
├── deployment.md                      # 部署说明（高层）
├── polymarket_repos_analysis*.md      # Polymarket 相关仓库调研与对比
├── 跟单猫技术选项文档.md             # 外部调研拼接稿（原始材料，未经契约化）
└── 跟单猫技术选项文档-梳理与建议.md  # 基于代码现状的梳理、纠偏与落地路线图
```

## 依赖关系（文档到代码）

- `docs/跟单猫技术选项文档-梳理与建议.md`：对照 `services/copy-trading-service/`，输出可执行改造清单；任何跟单服务核心行为变更都应同步更新该文档。

## 维护原则

- 原始材料与“可执行规格”分离：原文保留，但需要另写一份契约化版本（字段、时序、异常、回放/测试方法）。
- 任何新增/变更核心流程（监听/风控/下单/密钥）必须同步更新相关文档，否则等同于系统失忆。
